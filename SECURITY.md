# Security Policy

## Reporting a vulnerability

Please do **not** open a public GitHub issue for credential leaks, code execution,
supply-chain, or other security-sensitive vulnerabilities.

Report security issues privately to:

- roberto.delprete@esa.int

Use the subject line: `[sarpyx security] <short summary>`.

Please include:

- affected version or commit, if known;
- a concise description of the issue;
- reproduction steps or a proof of concept, if safe to share;
- any suggested mitigation.

General bugs, feature requests, and non-sensitive installation problems should be opened
as normal GitHub issues.
